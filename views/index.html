Exercise tracker

Create a New User

POST /api/exercise/new-user

Add exercises

POST /api/exercise/add

GET users's exercise log: GET /api/exercise/log?{userId}[&from][&to][&limit]

{ } = required, [ ] = optional

from, to = dates (yyyy-mm-dd); limit = number

See below for sample usage and outputs...

Possible Outputs:

  • When you GET /api/exerise/users, which retrieves a list of users -- [{"_id":"lhlVEAHjg","username":"test"},{"_id":"PJ-kCYiaq","username":"tom"}]
  • If you try to add a user that is already in the database -- {"userexists":"user already exists in the database"}
  • If you try to add activity log to a non existence user ID -- {"error":"User does not exist, create a new user first!"}
  • When you add a new activity log to a existing user ID -- {"username":"tom","description":"gym","duration":"60","_id":"PJ-kCYiaq","date":"Mon Mar 18 2019"}
  • When you retrieve all the activities log associating to a user ID -- {"totalExerciseMinutes":370,"userID":"PJ-kCYiaq","logs":[{"username":"tom","description":"run","duration":"40","date":"Mon Mar 18 2019"},{"username":"tom","description":"run","duration":"90","date":"Mon Mar 18 2019"},{"username":"tom","description":"run","duration":"90","date":"Mon Mar 12 2018"},{"username":"tom","description":"gym","duration":"90","date":"Thu Mar 15 2018"},{"username":"tom","description":"gym","duration":"60","date":"Mon Mar 18 2019"}]}
  • etc...

Example of usage:

  • 127.0.0.1:50023/api/exercise/users -- retrieves a list of users and their IDs
  • 127.0.0.1:50023/api/exercise/log?userId=PJ-kCYiaq -- retrieves all the logs associating to the user named tom
  • 127.0.0.1:50023/api/exercise/log?userId=PJ-kCYiaq&limit=2 -- retrieves all the logs associating to the user named tom and limits the results returned to 1
  • 127.0.0.1:50023/api/exercise/log?userId=PJ-kCYiaq&from=1990-02-23&to=2019-12-23 -- retrieves all the logs associating to the user named tom in the time span of 1990-02-23 to 2019-12-23
  • 127.0.0.1:50023/api/exercise/log?userId=PJ-kCYiaq&from=1990-02-23&to=2019-12-23&limit=3 -- retrieves all the logs associating to the user named tom in the time span of 1990-02-23 to 2019-12-23 and limits the number of results returned to 1
  • etc...
